# N皇后问题算法大比拼

## 问题描述

N皇后问题是一个经典的约束满足问题，需要在N×N的棋盘上放置N个皇后，使得任意两个皇后都不能互相攻击。根据国际象棋规则，皇后可以横向、纵向和对角线攻击。

### 数学背景

- **问题规模**：在N×N棋盘上需要放置N个皇后
- **约束条件**：每行、每列和每条对角线最多只能有一个皇后
- **解的数量**：有效解的数量随N变化：
  - N=4：2个解
  - N=8：92个解
  - N=12：14,200个解

### 算法复杂度
- **搜索空间**：N!（阶乘）种可能的排列
- **回溯算法**：通过剪枝显著减少搜索空间
- **时间复杂度**：最坏情况下O(N!)，但由于剪枝实际性能要好得多

## 实现的算法

### 1. DFS求所有解 (dfs_all)
- **方法**：标准回溯算法配合深度优先搜索
- **策略**：系统地探索所有可能的皇后放置方案
- **优化**：检测到冲突时早期剪枝
- **适用场景**：需要找到所有有效配置时

### 2. DFS求单个解 (dfs_single)
- **方法**：修改版DFS，找到第一个有效解即停止
- **策略**：找到第一个解后立即返回
- **优化**：无需探索找到解后的剩余分支
- **适用场景**：任意一个有效配置就足够时

### 3. BestFS求所有解 (bestfs_all)
- **方法**：使用启发式评估的最佳优先搜索
- **启发式**：基于冲突数量的棋盘状态评估
- **策略**：优先探索冲突较少的棋盘状态
- **优化**：使用优先队列按最有希望的状态排序探索
- **适用场景**：使用智能搜索顺序寻找所有解

### 4. BestFS求单个解 (bestfs_single)
- **方法**：为单个解优化的最佳优先搜索
- **启发式**：与BestFS-All相同的基于冲突的评估
- **策略**：找到第一个有效解后立即停止
- **优化**：结合启发式引导和早期终止
- **适用场景**：使用智能搜索快速找到单个解

## 算法比较

### 性能特征

| 算法 | 时间复杂度 | 空间复杂度 | 解类型 | 最佳使用场景 |
|------|------------|------------|--------|--------------|
| DFS-All | O(N!) | O(N) | 所有解 | 完整枚举 |
| DFS-Single | O(N!) | O(N) | 首个解 | 快速获取单个解 |
| BestFS-All | O(N!) | O(N²) | 所有解 | 引导式完整搜索 |
| BestFS-Single | O(N!) | O(N²) | 首个解 | 最快速获取单个解 |

### 关键差异

1. **搜索策略**：
   - DFS：深度优先探索，系统但可能较慢
   - BestFS：使用启发式引导搜索，可能更快

2. **内存使用**：
   - DFS：最小内存（O(N)递归栈）
   - BestFS：较高内存（O(N²)优先队列）

3. **解的发现**：
   - 所有解版本：找到所有可能的解
   - 单个解版本：在找到第一个有效解时停止

## 算法优缺点

### DFS-All
**优点：**
- 实现简单
- 内存使用最小
- 保证找到所有解
- 性能可预测

**缺点：**
- 可能探索不必要的分支
- 没有智能搜索排序
- 对于大N可能很慢

### DFS-Single
**优点：**
- 快速找到首个解
- 内存占用低
- 易于理解和调试
- 适用于中小棋盘

**缺点：**
- 不能保证解的质量（找到的第一个）
- 无法控制解的特性
- 可能错过更好的解

### BestFS-All
**优点：**
- 智能搜索排序
- 可能更快发现解
- 优先探索有希望的棋盘状态
- 更好的平均情况性能

**缺点：**
- 更高内存使用
- 实现更复杂
- 维护优先队列的开销
- 启发式不一定总是最优

### BestFS-Single
**优点：**
- 单个解最快
- 智能搜索引导
- 速度和智能的良好平衡
- 早期终止优化

**缺点：**
- 内存开销
- 启发式计算成本
- 可能找不到最优解
- 实现复杂

## 预期结果

对于不同棋盘尺寸，预期：

- **N=4-8**：所有算法表现良好，由于开销DFS可能更快

- **N=14+**：随着搜索空间指数级增长，BestFS变得更有价值
