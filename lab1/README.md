# 排序算法大比拼

## 项目简介

本项目实现了三种基于分治策略的经典排序算法：归并排序、快速排序和堆排序。通过性能测试和可视化演示，展示了不同算法在各种数据场景下的效率差异。

## 实现算法

### 1. 归并排序 (Merge Sort)
- **时间复杂度**: O(n log n)
- **空间复杂度**: O(n)
- **稳定性**: 稳定
- **实现方式**: 递归和迭代两种版本
- **特点**: 分治策略，将数组分成两半分别排序后合并

### 2. 快速排序 (Quick Sort)
- **时间复杂度**: 平均 O(n log n)，最坏 O(n²)
- **空间复杂度**: O(log n)
- **稳定性**: 不稳定
- **实现方式**: 递归和迭代两种版本
- **特点**: 选择基准元素进行分区，三数取中优化

### 3. 堆排序 (Heap Sort)
- **时间复杂度**: O(n log n)
- **空间复杂度**: O(1)
- **稳定性**: 不稳定
- **实现方式**: 基于二叉堆结构
- **特点**: 原地排序，空间复杂度最优

## 性能特点分析

### 不同场景下的算法表现

1. **随机数据**
   - 快速排序通常表现最佳
   - 归并排序性能稳定
   - 堆排序空间效率最高

2. **已排序数据**
   - 归并排序保持 O(n log n) 性能
   - 快速排序可能退化到 O(n²)
   - 堆排序性能相对稳定

3. **逆序数据**
   - 归并排序性能稳定
   - 快速排序性能可能显著下降
   - 堆排序表现稳定

4. **部分有序数据**
   - 归并排序表现优秀
   - 快速排序性能取决于基准选择
   - 堆排序性能相对稳定

5. **重复数据**
   - 归并排序稳定可靠
   - 快速排序可能因重复元素影响性能
   - 堆排序性能相对稳定

## 算法选择建议

### 选择归并排序当：
- 需要稳定的排序结果
- 数据量较大
- 内存空间充足
- 对最坏情况性能有要求

### 选择快速排序当：
- 追求平均情况下的最佳性能
- 内存空间有限
- 不要求排序稳定性
- 数据分布相对随机

### 选择堆排序当：
- 内存空间非常有限
- 要求原地排序
- 对最坏情况性能有要求
- 不要求排序稳定性
